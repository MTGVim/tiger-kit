# plan

## 목적

PRD, 티켓, 문서 같은 원천 자료가 아직 없을 때 사용자의 아이디어를 검토 가능한 브리프로 구체화합니다.

사용자가 “아직 자료는 없지만 같이 정리하자”, “인터뷰로 요구사항을 뽑자”, “계획 모드로 먼저 생각하자” 같은 요청을 할 때 이 스킬을 사용합니다.

## 동작 방식

사용자에게 보이는 응답과 TIGAP 산출물은 항상 한글로 작성합니다. 인용한 원문, 코드, 명령어, 파일 경로, 식별자는 원문 그대로 유지할 수 있습니다.

구현보다 질문, 정리, 계획을 우선합니다.

사용자가 명시적으로 구현까지 요청하지 않는 한 프로덕션 코드를 수정하지 않습니다.

## 필수 입력

원천 자료가 없어도 시작할 수 있습니다.

대신 최소한 다음 질문으로 문제를 구체화합니다.

```text
- 해결하려는 문제는 무엇인가요?
- 원하는 완료 상태는 무엇인가요?
- 반드시 지켜야 할 제약이나 제외할 범위가 있나요?
- 참고해야 할 기존 구현, 문서, 화면, 이슈가 있나요?
- 성공 여부를 어떻게 검증할 수 있나요?
```

이미 설치된 planning 스킬을 쓰고 싶다면 사용자가 직접 지정하거나 선택하도록 합니다. 임의로 외부 스킬을 호출하지 않습니다.

## 산출물 디렉터리

가능하면 현재 git 브랜치 이름을 사용합니다.

산출물은 다음 경로 아래에 저장합니다.

```text
.gap/{branch_name}/
```

현재 브랜치가 `main`, `master`, `develop` 또는 저장소 기본 브랜치라면 산출물 작성 전에 원천 자료에 맞는 작업 브랜치나 명시적 작업 ID를 권유합니다. 사용자 승인 없이 브랜치 생성이나 전환을 수행하지 않습니다.

## 출력 파일

사용자의 답변이 원천 자료로 충분히 정리되면 다음 파일을 생성하거나 갱신할 수 있습니다.

```text
.gap/{branch_name}/normalized/source-packet.md
```

구현 방향까지 충분히 구체화되면 다음 파일을 생성하거나 갱신할 수 있습니다.

```text
.gap/{branch_name}/plan/implementation-plan.md
```

계획 검토가 끝나기 전에는 실행 작업 목록을 확정하지 않습니다.

## 진행 절차

### 1. 문제 인터뷰

사용자에게 문제, 목표, 제약, 범위 제외, 검증 방법을 묻습니다.

답변이 부족하면 추측하지 말고 추가 질문을 합니다.

### 2. 브리프 정리

사용자 답변을 다음 항목으로 정리합니다.

- 문제
- 목표
- 확인된 요구사항
- 인수 조건
- 제약
- 범위 제외
- 열린 질문
- 검증 방법

### 3. 현재 상태 확인

사용자가 코드 경로나 기존 구현을 언급한 경우에만 관련 파일을 확인합니다.

자료 없이 저장소 전체를 임의 분석하지 않습니다.

### 4. 다음 단계 제안

브리프가 원천 자료로 충분하면 `/tigap:gap`으로 넘길 수 있다고 안내합니다.

구현 방향이 충분히 명확하면 `/tigap:gaplan`으로 구현 계획 초안을 만들 수 있다고 안내합니다.

## 완료 기준

이 스킬은 다음 조건을 만족하면 완료됩니다.

- 문제와 목표가 한글로 정리됨
- 확인된 요구사항과 열린 질문이 분리됨
- 검증 방법이 적어도 하나 이상 있음
- 다음 단계가 `/tigap:gap` 또는 `/tigap:gaplan` 중 하나로 명확함

## 인계

브리프가 준비되면 다음 중 하나를 추천하며 마무리합니다.

```text
/tigap:gap
/tigap:gaplan
```
