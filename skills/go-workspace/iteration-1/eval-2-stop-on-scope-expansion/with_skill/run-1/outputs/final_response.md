현재 요청은 여기서 멈춰야 합니다.

`tigap:go`는 `.gap/{branch_name}/tasks.md`의 첫 번째 `Ready` 작업 하나만 좁게 실행하는 단계입니다. 그런데 요청에 “다음 Ready 작업을 하다가 인증 구조 전체 리팩터링이 필요해 보이면 같이 해줘”라고 되어 있어, 선택한 Ready 작업 하나를 넘어서는 전체 인증 구조 리팩터링까지 범위가 확장될 수 있습니다.

스킬의 중단 조건에 따라 다음 상황에서는 구현을 진행하지 않고 보고해야 합니다.

- 작업 범위를 늘려야 함
- 선택한 Ready 작업 하나를 넘는 리팩터링이나 추가 구현이 필요함
- 구현이 관련 없는 영역에 영향을 줌

따라서 이번 실행에서는 인증 구조 전체 리팩터링을 함께 진행하지 않겠습니다. 먼저 다음 중 하나로 범위를 정해 주세요.

1. 기존 `Ready` 작업 하나만 좁게 진행
2. 인증 구조 전체 리팩터링을 새 갭/계획 작업으로 분리하기 위해 `/tigap:gap` 또는 `/tigap:gaplan` 실행
3. 현재 작업을 `Blocked`로 두고, 인증 리팩터링 필요성을 별도 후속 작업으로 추가

또한 현재 브랜치가 `main`이므로, 실제 코드 변경이나 TIGAP 산출물 갱신을 시작하기 전에는 전용 작업 브랜치 또는 명시적 작업 ID로 진행하는 것을 권장합니다. 사용자 승인 없이 브랜치 생성이나 전환은 하지 않겠습니다.

진행률: 이번 평가는 가상 상태이며 `.gap` 파일을 읽거나 수정하지 않도록 지시되었으므로 실제 `tasks.md` 기준 진행률은 계산하지 않았습니다.
