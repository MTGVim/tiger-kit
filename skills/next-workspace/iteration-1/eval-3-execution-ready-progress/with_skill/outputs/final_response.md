현재 단계: `execution-ready`

- 현재 브랜치/작업 ID 맥락: 제공된 가상 상태 기준이며, 실제 브랜치나 작업 ID는 확인하지 않았습니다.
- 기반 브랜치 여부: 확인 불가입니다. 다음 행동이 변경 가능한 작업이므로 현재 맥락이 `main`, `master`, `develop` 같은 기반 브랜치라면 전용 작업 브랜치나 명시적 작업 ID를 먼저 정하는 것을 권장합니다.
- 근거 산출물: `tasks.md`가 있고 실행 가능한 Ready 작업이 남아 있습니다.
- 진행률: Done 2개, In Progress 0개, Ready 3개, Blocked 1개입니다.
- 다음 작업: In Progress가 없으므로 첫 번째 Ready 작업을 다음 작업으로 진행하면 됩니다.
- 다음 추천 명령: `/tigap:go`

참고로 Blocked 작업이 1개 있지만 Ready 작업이 3개 있으므로 전체 상태는 `blocked`가 아니라 `execution-ready`로 판단합니다.
