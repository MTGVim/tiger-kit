현재 TIGAP 단계는 **시작 전 / 원천 자료 필요 단계**입니다.

`.gap/current/`에 산출물이 없다고 가정하면, 아직 다음 파일들이 만들어지지 않은 상태입니다.

- `normalized/source-packet.md`
- `analysis/gap-report.md`
- `plan/implementation-plan.md`
- `tasks.md`
- `execution-log.md`
- `review-checklist.md`

따라서 지금은 구현이나 작업 실행 단계가 아니라, 먼저 기준이 될 원천 자료를 확보해야 합니다.

다음 행동 중 하나를 선택하면 됩니다.

1. 요구사항, 이슈, 문서, 사용자 피드백 등 원천 자료를 제공한다.
2. 저장소나 특정 파일에서 요구사항을 추출하라고 지시한다.
3. 원천 자료가 아직 없다면 인터뷰 방식으로 목표와 범위를 정리한다.

읽기 전용 확인만 필요한 상황이므로 `.gap` 파일 생성, 브랜치 변경, git 상태 변경은 하지 않는 것이 맞습니다.
