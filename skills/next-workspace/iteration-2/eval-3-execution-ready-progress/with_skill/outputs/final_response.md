현재 단계: `execution-ready`

현재 브랜치/작업 ID 맥락: 현재 브랜치는 `main`입니다. 별도 작업 ID 맥락은 입력에서 제공되지 않았습니다.

기반 브랜치 여부: `main`은 기반 브랜치로 보입니다. 다음 행동이 `/tigap:go`처럼 변경 가능한 작업이므로, 실행 전에 전용 작업 브랜치나 명시적 작업 ID를 먼저 정하는 것을 권장합니다. 제가 브랜치를 만들거나 전환하지는 않았습니다.

근거 산출물:
- `tasks.md`가 있다고 가정한 상태입니다.
- 작업 상태가 Done 2개, In Progress 0개, Ready 3개, Blocked 1개입니다.

진행률:
- Done: 2
- In Progress: 0
- Ready: 3
- Blocked: 1

판정 근거: In Progress 작업은 없고 Ready 작업이 3개 있으므로, 해결되지 않은 Blocked 작업이 1개 있더라도 바로 실행 가능한 작업이 남아 있어 `execution-ready` 단계입니다.

다음 작업: 첫 번째 Ready 작업을 다음 작업으로 진행하면 됩니다. 구체적인 작업명은 입력에 없으므로, 실제 `tasks.md`에서 가장 위에 있는 Ready 항목을 선택하세요.

다음 추천 행동:
- 기반 브랜치인 `main`에서 바로 변경 작업을 시작하지 말고, 먼저 전용 작업 브랜치나 작업 ID를 정하세요.
- 이후 `/tigap:go`로 Ready 작업 하나만 좁게 실행하세요.

안전 확인: 이번 확인은 읽기 전용으로 수행했으며, `tasks.md`를 생성·수정·삭제하지 않았고 git 상태도 변경하지 않았습니다.
