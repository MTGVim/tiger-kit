# 동작 검증

실제 클릭, 입력, 키보드, 포커스, 제출, 대화 상자 경로를 실행하세요. 합성 이벤트만으로 성공을 주장하지 말고 사용자에게 보이는 상태 전환과 오류·차단 상태를 관찰하세요.

변경 작업은 UI transition, 관련 network request/response, 최종 UI 상태를 하나의 흐름으로 연관 지으세요. 요청이나 응답만, 또는 토스트·국소 DOM 변경만으로는 충분하지 않습니다. 모든 성공·실패·차단 최종 상태는 screenshot을 저장하고 그 이미지 자체를 분석해야 합니다. 필수 캡처나 분석이 없으면 `Pass`가 아니라 `Unverifiable`입니다.
