# Review Checklist

## Requirements

- [ ] Requirements are mapped to implementation.
- [ ] Non-goals were not accidentally implemented.
- [ ] Ambiguous items are documented.

## Code

- [ ] Existing patterns were followed.
- [ ] Changes are scoped to the task.
- [ ] Naming is consistent.
- [ ] Error states are handled.
- [ ] Edge cases are covered where relevant.

## Validation

- [ ] Typecheck passed or reason recorded.
- [ ] Lint passed or reason recorded.
- [ ] Tests passed or reason recorded.
- [ ] Build passed or reason recorded.

## Review Notes

- [ ] Risky changes are called out.
- [ ] Follow-up tasks are documented.
