description: 원천 자료와 현재 구현 사이의 갭을 분석합니다.
---

이 플러그인의 `gap` 스킬을 사용합니다.

사용자에게는 한글로 답합니다. TIGAP 산출물도 한글로 작성합니다. 단, 인용한 원문, 코드, 명령어, 파일 경로, 식별자는 원문 그대로 유지할 수 있습니다.

목표: 원천 자료를 수집하고, `.gap/{branch_name}/normalized/source-packet.md`로 정규화하고, 가능한 범위에서 현재 구현을 확인한 뒤 `.gap/{branch_name}/analysis/gap-report.md`를 작성합니다.

사용자가 원천 자료나 자료 추출 지시를 제공하지 않았다면 저장소를 확인하거나 산출물을 쓰기 전에 멈춥니다. 어떤 원천 자료가 필요한지 설명하고, 사용자가 자료를 직접 제공하거나, 자료를 가져오거나 추출하라고 지시하거나, 인터뷰를 시작하도록 요청합니다.

변경 가능한 산출물을 쓰기 전에 브랜치/작업 ID 맥락을 확인합니다. 현재 브랜치가 `main`, `master`, `develop` 또는 저장소 기본 브랜치라면 원천 자료에 맞는 작업 브랜치나 명시적 작업 ID를 권장합니다. 사용자 승인 없이 브랜치를 만들거나 전환하지 않습니다.

명시적으로 요청받지 않는 한 이 명령에서는 코드를 구현하지 않습니다.
